Report an issue

Where to report

You can report issues in two ways:

  • File an issue on GitHub at github.com/subpop/Relay/issues
  • Message the maintainers in #relayapp:matrix.org

What to include

A good bug report includes:

  • A clear description of what happened and what you expected
  • Steps to reproduce the issue
  • Your homeserver hostname (e.g., matrix.example.org)
  • Whether other Matrix clients (Element X, Element Web) show the same behavior
  • An Activity Log export filtered to the relevant category (see Use the Activity Log)

Export the Activity Log

Open Window > Activity Log (⌥⌘A), filter to the relevant category, and press ⌘S to save.

Privacy considerations

The Activity Log export contains room IDs, device IDs, and your homeserver hostname. It does not contain message content, encryption keys, or authentication tokens. If you prefer not to share this information publicly, contact the maintainers privately in the Matrix room.

Tip: The more specific your bug report, the faster it can be diagnosed. "Calls don't work" is harder to investigate than "Audio calls connect but no sound is heard when calling from @user:example.org to @other:matrix.org."
